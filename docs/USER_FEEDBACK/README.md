# User Feedback Repository

This directory contains user feedback, interview notes, and validation results.

## Structure

- `YYYY-MM-DD-interview-[user-id].md` - Individual interview notes
- `feedback-summary.md` - Aggregated feedback patterns
- `action-items.md` - Prioritized improvements from feedback
- `validation-results.md` - Experiment results and learnings

## Interview Template

```markdown
# User Interview - [Date]

**User ID:** [user-id]
**Interviewer:** [name]
**Duration:** [minutes]

## Background
- Role:
- Experience:
- Use case:

## Questions & Answers
1. Q: [question]
   A: [answer]

## Key Insights
- [insight 1]
- [insight 2]

## Action Items
- [ ] [action item]
```

## Feedback Collection Process

1. Conduct interview using `/docs/user-interview-guide.md`
2. Create interview notes file
3. Update `feedback-summary.md` with patterns
4. Add action items to `action-items.md`
5. Update validation results if applicable
