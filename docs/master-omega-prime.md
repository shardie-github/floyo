## Quick Start

```bash
# Diagnostic mode (read-only)
npm run master-omega-prime

# Auto-fix mode (applies fixes)
npm run master-omega-prime:fix

# Run specific phase
npm run master-omega-prime -- --phase=3

# Verbose output
npm run master-omega-prime -- --verbose
```

## 10-Phase Execution Flow

### PHASE 1 ‚Äî Stack Detection & System Diagnostics

Detects which stack components are present and performs a full system health check:

- Environment variables validation
- GitHub Secrets alignment
- Branch alignment
- Supabase schema drift detection
- Edge function status
- RLS correctness
- Vercel project link health
- Expo EAS config
- Shopify integration status
- TikTok/Meta API linkage
- Zapier flows
- Google Sheets sync

**Output:** `STACK STATUS REPORT` with components detected, gaps found, and fixes applied.

### PHASE 2 ‚Äî Self-Healing & Auto-Repair

For every detected mismatch, generates:

- Code patches
- Workflow updates
- Schema migrations
- CI/CD fixes
- Env alignment fixes
- Cross-platform linkage fixes

Then automatically re-runs diagnostics to confirm resolution.

**Output:** List of fixes applied and remaining issues.

### PHASE 3 ‚Äî Backend Orchestration (Supabase + Prisma)

Performs:

- Schema introspection
- Missing table/function/index creation
- RLS validation
- Type generation
- Prisma WASM validation
- Edge function deployment
- API route health check
- Migrations fix/generation

Ensures no duplication, only creates missing entities.

**Output:** `BACKEND HEALTH REPORT` with tables verified, RLS verified, functions deployed, schema drift resolved.

### PHASE 4 ‚Äî Frontend Deployment (Vercel + Expo)

**Vercel:**
- Verifies Vercel linkage
- Checks branch ‚Üí production mapping
- Validates envs
- Triggers new deploy (if configured)
- Confirms deployed build matches latest commit

**Expo:**
- Validates Supabase keys
- Validates OTA channels
- Dry-run EAS build
- Corrects config drift
- Patches app.json / eas.json if required

**Output:** `DEPLOYMENT REPORT` with Vercel and Expo status.

## GitHub Actions Integration

Add to `.github/workflows/master-omega-prime.yml`:

```yaml
name: Master Omega Prime

on:
  schedule:
    - cron: '0 0 * * *'  # Daily
  workflow_dispatch:

jobs:
  orchestrate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm run master-omega-prime:fix
        env:
          SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
          SUPABASE_ANON_KEY: ${{ secrets.SUPABASE_ANON_KEY }}
          # ... other secrets
```

## Troubleshooting

### "Missing environment variables"

Set the required GitHub Secrets:
- `SUPABASE_URL`
- `SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `VERCEL_TOKEN`
- `EXPO_TOKEN`

### "Schema drift detected"

Run Supabase migrations:
```bash
npm run supa:migrate:apply
```

### "Vercel deployment failed"

Check Vercel project linkage:
```bash
npm run vercel:pull
```

## Best Practices

1. **Run diagnostics first**: Always run without `--fix` first to see what needs attention
2. **Review fixes**: Check the generated fixes before applying in production
3. **Incremental execution**: Use `--phase=N` to run specific phases
4. **Regular execution**: Run daily via GitHub Actions for continuous health monitoring
5. **Combine with Aurora Prime**: Run both orchestrators for complete coverage
