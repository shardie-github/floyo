# Support

## Getting Help

### GitHub Issues
- **Bugs**: Open an issue with the `bug` label
- **Feature Requests**: Open an issue with the `enhancement` label
- **Questions**: Open an issue with the `question` label

### Documentation
- Check the [README.md](README.md) for quick start guides
- Review [docs/](docs/) for detailed documentation
- See [DEVELOPER_GUIDE.md](docs/DEVELOPER_GUIDE.md) for development setup

## Reporting Issues

When reporting issues, please include:
1. **Description**: Clear description of the problem
2. **Steps to Reproduce**: If applicable
3. **Expected Behavior**: What should happen
4. **Actual Behavior**: What actually happens
5. **Environment**: OS, Python/Node versions, dependencies
6. **Logs**: Relevant error messages or logs

## Emergency Support

For critical production issues:
1. Tag maintainers in the issue
2. Use the `critical` label
3. Provide as much context as possible

## Community

- **Discussions**: Use GitHub Discussions for general questions
- **Contributions**: See [CONTRIBUTING.md](CONTRIBUTING.md) if you want to contribute

## Response Times

- **Critical Issues**: 24-48 hours
- **Bug Reports**: 3-5 business days
- **Feature Requests**: Reviewed during planning cycles
- **Questions**: Community-driven, maintainers respond when available
